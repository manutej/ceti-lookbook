02 — What it can actually do

Six surfaces it touches today.

Computer use isn't general intelligence — it's a tight, repeatable loop of screenshot → reason → click/type → screenshot. These are the surfaces where that loop already pays for itself.

FORM FILLING

Long forms, every field, no copy-paste.

Vendor onboarding, expense reports, insurance claims, government portals. The agent reads the source data, the destination form, and stitches the fields in order — even when the labels don't match.

04 — The honest part

Safety isn't a feature — it's the shape of the deployment.

A browser agent that can click anywhere can also be tricked into clicking anywhere. The mitigation isn't a clever prompt — it's the same defense-in-depth you'd give a junior contractor with your laptop password.

Three boundaries to draw before shipping

When Claude can take screenshots, move your mouse, and type on your keyboard, any prompt-injection vulnerability becomes an OS-level vulnerability. A poisoned webpage, a malicious email preview, an attacker-controlled PDF — each can hijack an unscoped agent. The OWASP Top 10 for Agentic Applications 2026 formalized five categories: prompt injection, resource exhaustion, data exfiltration, lateral movement, and tool misuse.

None of this means "don't ship." It means the agent runs in a sandbox you trust, with credentials scoped to the task, on a network you control. The vendors above expose those primitives — using them is on you.

Isolation Containers (cheap, kernel-shared), gVisor (~15% overhead), Firecracker microVMs (150ms cold start), Kata, WASM. Pick by threat model.
Credential scope Short-lived tokens via a credential broker. Don't paste long-lived API keys into a prompt the model can leak.
Network & FS Egress allowlist (proxy/IP/port). Write operations confined to the workspace. Manual approval for net-new connections.

05 — What it looks like in code

A minimum viable computer-use call.

The loop is small. The model returns tool calls, your harness executes them on the sandboxed VM, you send the next screenshot back. Repeat until the task is done.

computer_use_example.py python · anthropic SDK 
from anthropic import Anthropic

client = Anthropic()

response = client.beta.messages.create(
    model="claude-opus-4-5",
    max_tokens=1024,
    tools=[{
        "type": "computer_20251124",   # enhanced actions
        "name": "computer",
        "display_width_px": 1280,
        "display_height_px": 800,
    }],
    messages=[{"role": "user", "content": "Open the inbox and draft a reply to Priya."}],
    betas=["computer-use-2025-11-24"],
)
# → response.content contains tool_use blocks: click, type, screenshot, zoom…
# Execute them in your sandbox, send the next screenshot back, loop.
