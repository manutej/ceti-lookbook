The agent loop

A thinking, acting, observing cycle — with memory.

Every LangGraph agent is a graph. Nodes do the work. Edges decide what runs next. State threads through all of it — so the agent doesn't restart its brain on every step.

The ReAct loop, drawn as a graph.

Each node receives the full state, returns an update, and hands control to the next node. The pulse you see is one tick of the agent — in production it might be a single LLM call, a tool invocation, and an observation.

  1. ReasonThe LLM reads the conversation so far and decides what to do next — or whether it's done.
  2. ActThe agent calls a tool: a search, a database query, a Python function, an external API.
  3. ObserveThe tool's output is appended to state and the loop continues — or routes to END.

Why a graph — not a chain

Chains run forward. Graphs remember, branch, pause, resume.

01 · State

Memory is a first-class type.

Define a typed state schema once. Every node reads it, returns updates, and the graph merges them. No more passing context dictionaries through ten function calls.

02 · Routing

Conditional edges.

A node returns "search" or "respond" — the graph routes accordingly.

03 · Durability

Checkpoint anywhere.

Every step persists. Crash a node, restart from the last good state.

04 · Oversight

Pause for a human, then resume.

Inspect state mid-run. Approve a tool call. Edit a draft. The graph waits, then continues from exactly where it stopped — minutes or days later.

What it looks like in code

Ten lines to a working stateful agent.

Python. No magic. Define a state, add nodes, draw the edges, compile.

def reason(state: MessagesState):
    # LLM looks at messages, decides next step

Multi-agent

Specialist teams of agents

A supervisor node routes work to a researcher, a writer, a critic. Each is its own subgraph. State threads through all of them.

Conditional flow

Branching on what the model said

If the answer mentions a SQL query → run it. If it asks for clarification → route to a human. The graph encodes the decision.

Long-running

Workflows that span days

An agent drafts an email Monday, waits for approval Tuesday, sends it Wednesday. Checkpointing makes that one logical run.
