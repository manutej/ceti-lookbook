The pipe operator

One vertical bar. Five stages. Anything in between.

LangChain Expression Language (LCEL) borrows the Unix pipe: write prompt | retriever | format | model | parser and you have a working chain. Each piece implements the same Runnable contract, so order, swap, and reuse are free.

The | operator isn't a string-template hack. It's Python's __or__ overloaded so every Runnable composes with every other Runnable — and the result is itself a Runnable. Sequences compose into sequences compose into pipelines.

That single property is why LCEL chains support streaming, batching, async, and tracing for free. You write one definition; LangChain gives you four execution modes without a second line of code.

The Runnable contract

Four verbs. Same pipeline.

Every piece of an LCEL chain exposes the same four methods. Write your pipeline once and pick how it runs based on what you need.

.invoke()

One question, one answer.

A single synchronous call: pass an input, wait for the result. The default mode for prototypes, scripts, and CLI tools.

.batch()

Many inputs in parallel.

Run the same pipeline across a list of inputs concurrently — the I/O-bound ones overlap, latency stays flat instead of multiplying.

.stream()

Token by token, as it happens.

Return the answer as soon as the model starts producing it. Same chain, same code — just a different verb. Chat UIs depend on this.

.ainvoke()

Async for everything else.

The same call, async — for web servers, agents, or anywhere you can't afford to block. Each verb has an a* twin.

What it looks like

Ten lines. A real RAG chain.

This is the entire pipeline: take a question, pull related docs, format both into a prompt, send to the model, parse the result. Every | is a hand-off.

Swap ChatOpenAI for ChatAnthropic? One line. Add a re-ranker? One pipe. Move from invoke to stream? Change .invoke() to .stream().

rag_chain.pyfrom langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_template(
    "Answer using only this context:\n{context}\n\nQ: {question}"
)
model  = ChatOpenAI(model="gpt-4o-mini")
parser = StrOutputParser()

# the whole pipeline — five components, four pipes
chain = {"context": retriever, "question": passthrough} | prompt | model | parser

chain.invoke("What's our return policy?")
