01 · Self-attention

Every token looks at every other token.

The 2017 idea, in one image: each token in the sequence asks "who else should I pay attention to?" The answer is a matrix of weights. Brighter cell = more attention from row→column.

02 · The layer cake

Stack the same block, 32 to 80 times.

A transformer is mostly one block, repeated. Tokens flow up through embeddings, through N identical layers of attention + feed-forward, and out as a probability distribution over the next token.

EMBED + POSin
ATTENTION + FFNL01
ATTENTION + FFNL02
ATTENTION + FFNL03
ATTENTION + FFN···
ATTENTION + FFNL80
UNEMBED → SOFTMAXout

N = 32–80 identical blocks in modern LLMs

embed

Token → vector

Each token becomes a learned ~4096-dimensional vector. Position information is added (sinusoidal in 2017, RoPE today).

attention

Mix across tokens

The Q/K/V machinery. The only place where tokens look at each other. Everything else acts per-token.

ffn

Think per token

A two-layer MLP applied to each token independently. ~⅔ of the model's parameters live here.

layer-norm

Keep the math sane

Normalizes activations so the signal doesn't explode or vanish across 80 layers. RMSNorm replaces LayerNorm in most modern stacks.

04 · What changed

Four upgrades that quietly redefined the block.

The shape of the transformer is the same. Inside the block, four ideas have become standard since 2020. Every frontier model in 2026 uses some combination of these.

positional

RoPE

Rotary positional embeddings. Encodes position by rotating Q/K vectors in complex space — generalizes cleanly to longer sequences than seen in training.

attention

GQA

Grouped-Query Attention. Multiple Q heads share one K/V pair — shrinks the KV-cache 4–8× with near-zero quality loss. The reason long-context inference is affordable.

sparsity

Mixture of Experts

Replace one big FFN with many small ones plus a router. Only a few experts activate per token — total params explode, compute per token stays flat.

systems

FlashAttention

Same math, better memory access. Fuses operations on-GPU to skip the slow HBM round-trip. Long context windows (128K+) became practical because of this.

05 · What it looks like in code

One equation, the whole idea.

The attention layer of every transformer reduces to this. Everything else — multi-head, layer-norm, residuals, FFN — wraps around it.

attention.py · scaled dot-product
# Q, K, V: (batch, heads, seq_len, d_k)
import torch
import torch.nn.functional as F

def attention(Q, K, V, mask=None):
    d_k = Q.size(-1)                          # key dim
    scores = Q @ K.transpose(-2, -1) / d_k**0.5  # QKᵀ / √d_k

    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)

    weights = F.softmax(scores, dim=-1)        # the matrix above
    return weights @ V                          # mix the values

That's it. Run it 32 times per layer (multi-head), stack the layer 80 times, and you have a frontier LLM. Everything since 2017 has been making this faster, cheaper, or longer.
