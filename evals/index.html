02 · Methodology

Four ways to know if a model is actually right.

No single eval method is enough. Mature teams layer all four — cheap automated checks on every request, judges on samples, humans on edges, A/B on rollouts.

01 · DETERMINISTIC

Ground-truth

A fixed test set with expected outputs. Compare with exact match, BLEU, ROUGE, or semantic similarity. Cheap, fast, repeatable — but only as good as your set.

Use for: regression suites, retrieval recall, structured outputs

02 · MODEL-GRADED

LLM-as-Judge

A second model scores the first’s output along axes like faithfulness, relevance, helpfulness. GPT-class judges reach ~80% agreement with human raters — same as human-to-human.

Use for: open-ended generation, RAG faithfulness, summary quality

03 · WETWARE

Human-in-loop

Domain experts grade a sliced sample weekly. The slowest signal, but the only one that catches subtle wrongness LLMs miss. Use it to calibrate your judge.

Use for: compliance, medical, legal, brand voice

04 · IN-PRODUCTION

A/B testing

Run two prompts, models, or chains side-by-side on real traffic. Measure on outcomes that matter — task completion, thumbs-up rate, deflection, revenue. The only eval that doesn’t lie.

Use for: prompt changes, model upgrades, retrieval tuning

04 · In code

Ten lines from zero to first eval.

A minimal LangSmith eval. Define a dataset, define a target (the thing under test), define an evaluator, run. The same shape generalizes to Braintrust, Arize Phoenix, and DeepEval.

eval.py # pip install langsmith openevals from langsmith import Client, evaluate from openevals.llm import create_llm_as_judge client = Client() dataset = client.read_dataset(dataset_name="support-qa-v3") def target(inputs: dict) -> dict: return {"answer": agent.invoke(inputs["question"])} faithfulness = create_llm_as_judge( prompt="Is the answer faithful to the retrieved context? 0–1.", model="openai:gpt-4.1", ) evaluate(target, data=dataset, evaluators=[faithfulness], experiment_prefix="v3-rerank")
