Pixels, text, sound — one model.

The frontier of 2026: image, text, and audio streams pass through modality-specific encoders, then meet in a shared embedding space. The model reasons across them as if they were one substance. This is what makes a screenshot "readable," a podcast "summarisable," and a video "queryable."

01 · The signature insight

Three streams. One embedding space. One reasoning loop.

Each modality has its own encoder — a vision transformer for image patches, a tokeniser for text, a Whisper-style encoder for audio frames. Their outputs land in the same vector space, where the language model treats them as just more tokens to attend to.

02 · What each modality unlocks

Four superpowers, one model.

Each modality changes the shape of problems the model can attack. The point isn't novelty — it's that the cost of switching tools (transcribe → summarise → caption → reason) drops to zero.

VISION

Read documents and screenshots.

Extract structured data from PDFs, parse UI screenshots, OCR receipts, understand charts. Image input ≈ width × height ÷ 750 tokens on Claude.

{ total: 42.50,
  tax: 3.50 }
~1334 tokens · 1000×1000 px
AUDIO

Transcribe and respond in voice.

Native audio in/out enables voice agents that hear tone, accent, and pauses — and reply with prosody. Latency drops from 2.8s (chained pipeline) to ~320ms.

25ms frames · prosody preserved
VIDEO

Summarise and search across frames.

Gemini 3 samples up to 10 FPS to catch fast actions; Claude 4 reads keyframes for narrative. Ask "where does the chart go up?" and get a timestamp.

10 FPS · multi-clip reasoning
UNIFIED REASONING

Cross the streams.

The real unlock isn't "vision OR audio OR text" — it's reasoning across them in the same prompt. Watch a tutorial video, listen to a customer call, paste a dashboard screenshot — and ask one question about all three. The model attends to every modality through the same self-attention layer.

04 · The shape of a multimodal call

One request. Two modalities. Same response shape.

The API surface is almost boring — and that's the point. An image is just another content block alongside text.

python · anthropic SDK 
import anthropic, base64, httpx

# Load an image as base64 — could equally be a local file
img_url = "https://example.com/receipt.png"
img_b64 = base64.standard_b64encode(httpx.get(img_url).content).decode("utf-8")

client = anthropic.Anthropic()
msg = client.messages.create(
    model="claude-opus-4-7",
    max_tokens=1024,
    messages=[{
        "role": "user",
        "content": [
            {"type": "image", "source": {
                "type": "base64",
                "media_type": "image/png",
                "data": img_b64,
            }},
            {"type": "text", "text": "Extract total, tax, and date as JSON."},
        ],
    }],
)
print(msg.content[0].text)
# → { "total": 42.50, "tax": 3.50, "date": "2026-05-15" }
