EMBEDDINGS · VECTOR DATABASES · 2026

Meaning, as coordinates.

An embedding is a paragraph compressed into 1,536 numbers — coordinates in a space where nearness equals meaning. A vector database is the filing cabinet that finds the nearest neighbors at a billion-point scale, in milliseconds.

Why this works. A neural network reads millions of sentences and learns to place words near each other in a high-dimensional space if they appear in similar contexts. The model never sees the word "dog" — it sees a list of 1,536 numbers. Two paragraphs about pets land closer together than a paragraph about pets and one about trucks. Distance is meaning.

Why a database. "Closest to" is cheap on a hundred vectors and ruinous on a billion. Vector databases use approximate-nearest-neighbor indexes (HNSW, IVF) to return the top-k matches in milliseconds — the engine behind every modern RAG system, semantic search, recommendation, and "find similar" feature.

FOUR THINGS TO KNOW

Distance metrics & index algorithms.

Picking a vector store is mostly picking these four levers. Get them wrong and your retrieval is slow, wrong, or both.

01 · Cosine

Cosine similarity

Measures the angle between two vectors, ignoring length. The default for text embeddings — a long document and a one-line query can still match perfectly.

02 · Euclidean

Euclidean (L2) distance

Straight-line distance between points. Useful when magnitude matters — image embeddings, geo, and any space where vectors aren't unit-normalized.

03 · Dot product

Dot product

Cheap, fast, and equivalent to cosine when vectors are normalized. The metric most production vector DBs run on the hot path.

04 · HNSW

Hierarchical Navigable Small World

The default index in nearly every modern vector DB. Builds a layered graph of vectors — log-time approximate search across billions of points with >95% recall.

END-TO-END IN 11 LINES

Embed once, query forever.

The full RAG primitive in plain Python — embed a corpus, push to a vector store, retrieve the top-3 most similar passages for any query. Substitute your provider; the shape is identical everywhere.

embeddings.py · openai + qdrant · py 3.12 
from openai import OpenAI
from qdrant_client import QdrantClient, models

oa = OpenAI()
qd = QdrantClient("http://localhost:6333")

# 1 — turn text into 1,536-dim vectors
docs = ["a dog wags its tail", "the truck hauls cement", "pasta needs salt"]
vecs = [e.embedding for e in oa.embeddings.create(model="text-embedding-3-small", input=docs).data]

# 2 — index them under a collection
qd.recreate_collection("corpus", vectors_config=models.VectorParams(size=1536, distance="Cosine"))
qd.upload_points("corpus", [models.PointStruct(id=i, vector=v, payload={"text": d}) for i, (v, d) in enumerate(zip(vecs, docs))])

# 3 — nearest-neighbor query in milliseconds
q = oa.embeddings.create(model="text-embedding-3-small", input=["my pet is happy"]).data[0].embedding
hits = qd.search("corpus", query_vector=q, limit=3)
print([h.payload["text"] for h in hits])  # → ["a dog wags its tail", ...]
