Cost delta

What changes the curve. The 10K-token system prompt is the dominant input cost — read on every request. Mark it cacheable and the first request pays the 1.25× write premium; every subsequent request within the TTL pays 0.1× read. Five reads recoup the write, then the discount compounds.

Why the shape is jagged. Each visible drop on the cached line is a 5-minute window where the cache stayed warm. The line nudges up between reads (live output tokens still cost full rate) and barely climbs during sustained traffic. At enterprise volume this is the difference between a $200/day bill and a $24/day bill — same product, same model.

Cost-optimization checklist

Five things engineers should always do.

In rough order of return. Most teams ship the first three on day one — the second two are where the savings compound.

01
Cache the system prompt.
Anything > 1024 tokens that doesn't change between requests — system instructions, tool definitions, few-shot examples, long document prefixes — should be cacheable.
−60 to −90%
02
Cap max_tokens aggressively.
Output is 5× input. Pick the smallest budget that still produces complete answers, then audit. Most production prompts ship with the default — and pay for it.
−20 to −40%
03
Route by complexity, not by default.
A small classifier (Haiku, $0.10/MTok cached) in front of Sonnet/Opus catches the 70% of requests that don't need the flagship model. Cost of routing is rounding error.
−40 to −70%
04
Move async work to the batch API.
Overnight evals, embedding backfills, summarization sweeps, scheduled report generation. 50% off, stacks with caching for ~95% combined savings on long prefixes.
−50% flat
05
Instrument cost per request, not just totals.
Tag every call with feature, model, and cache-hit ratio. The bill is your monthly aggregate — the cost-per-request histogram is what tells you where to optimize next.

In code

Prompt caching, ten lines.

Anthropic SDK · mark a block as cacheable with cache_control. The first call writes; every call within the 5-minute TTL pays 10%.

cache.py · Anthropic Python SDK 
from anthropic import Anthropic

client = Anthropic()

resp = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=800,
    system=[
        {
            "type": "text",
            "text": long_system_prompt,           # ~10K tokens, stable across turns
            "cache_control": {"type": "ephemeral"}  # 5-min TTL · "1h" also available
        }
    ],
    messages=[{"role": "user", "content": turn}],
)
print(resp.usage.cache_read_input_tokens, "tokens read from cache")
# First call: usage.cache_creation_input_tokens populated (1.25× cost)
# Subsequent calls within TTL: cache_read_input_tokens populated (0.1× cost)
