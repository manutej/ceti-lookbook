Section 02 · Three philosophies

The agents disagree about where coding happens.

Once you sort the field by deployment surface, the disagreement isn't about model quality — it's about who is supposed to be in the chair.

Philosophy A

IDE copilots

The agent lives inside the editor. The developer drives; suggestions arrive at the cursor and in side panels. Composer / Agent Mode adds multi-file work without leaving the IDE.

Cursor GitHub Copilot Codeium
Philosophy B

Terminal-first agents

The agent lives in your shell. The conversation is the workflow — repo as work surface, MCP servers as tools, edits applied as git diffs. Long-horizon tasks are first-class.

Claude Code Aider OpenAI Codex CLI
Philosophy C

Autonomous agents

The agent runs on a server. You file a ticket; it plans, browses, codes, tests, opens a PR. The developer reviews instead of types. Maximum leash length — and maximum failure surface.

Section 03 · Pick the right tool

Most teams use two of these. Rarely one. Never all.

A short decision table. Read it as "if your job today looks like the first column, the second column is a sensible default." Substitute according to budget and stack.

Use case
Sensible default
Why
Greenfield, fast iteration
Cursor
Tab completion + Composer keeps you in the editor while the codebase is small enough to hold in your head. Agentic work is still cheap.
Refactor a large codebase
Claude Code
Long-horizon planning, repo-wide context, terminal access, MCP servers for project-specific tools. The leash is intentionally long.
Debug a tricky failure
Cline
Every action is explicit and approval-gated. You can watch it form a hypothesis, run a test, revise. BYO key means model swaps are cheap.
Learning a new language
Aider
Aider's Polyglot benchmark exists because Aider is good at polyglot work. Git-native, transparent edits, zero subscription friction.
Ship a small ticket end-to-end
Devin
When the work is well-scoped and you'd rather review than write, the autonomous path is the cheapest in human-minutes. Pricier in dollars.
Drop-in for an existing team
GitHub Copilot
Zero adoption friction, every IDE, every language, an admin can roll it out across an org. Lowest peak power, highest baseline coverage.
