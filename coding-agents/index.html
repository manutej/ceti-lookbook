Section 02 · The Roster

Seven Players, Seven Team Colors

A league this young still has its founders on the field. Claude Code only shipped in early 2025, but in 2026 it sits at the top of the SWE-bench Verified table. Cursor took an obvious idea — fork VS Code, build the agent in — and made it the default IDE for a generation of new engineers. GitHub Copilot, the senior pro, was the first to make AI coding feel ordinary and remains the only player with serious enterprise distribution. Aider, the open-source veteran, is the workshop where the Polyglot benchmark was forged. Cline ran the autonomous loop inside the editor before "agent mode" was a marketing term. Devin pitched the autonomous junior engineer and forced the rest of the league to argue about what autonomy even means. Codeium ships the free tier that the other six can't price-match.

The IDE Battle · Which Editor Wins the Default Slot

Cursor is what happens when you build the editor around the agent instead of bolting an agent onto the editor. The tab-completion model is fine-tuned for accept-rates inside Cursor specifically, and Composer / Agent Mode lets the model touch a whole folder without you switching contexts. The price you pay is fork lock-in: your VS Code extension story sits behind whatever Anysphere's release cycle decides to support, and your CIO is now approving a non-Microsoft IDE.

Cursor's SWE-bench Verified score of 67.2% (with its own Auto router) sits comfortably mid-table, but the benchmark understates the value — most of Cursor's lift shows up in autocomplete keystrokes, not in agent runs. It is the league's clearest player development story: hires who learn on Cursor write more code, faster, sooner.

Copilot is the senior pro and the only player your enterprise security team has already approved. It ships inside every IDE that matters — VS Code, JetBrains, Visual Studio, Neovim, Xcode — and the admin console answers every compliance question. Agent Mode and MCP support landed in 2025–26, closing the gap with Cursor on agentic work, though the inline autocomplete model is still optimized for "junior pair on shoulder," not "senior pair driving."

The hidden Copilot advantage is the GitHub graph. Copilot Workspaces, PR summaries, code review, and security suggestions all share signal across an organization's repos. That lift doesn't show up in SWE-bench but it shows up on a Wednesday afternoon when you're reviewing a teammate's PR.

Pick When Choose Cursor for greenfield work, fast iteration, and teams that will adopt a non-Microsoft editor. Choose Copilot when your org needs one default that everyone gets and an admin can roll it out across thousands of seats in an afternoon. Most large engineering orgs end up running both — Copilot as floor, Cursor as ceiling.

The Terminal Battle · Which Shell Pair Wins Long-Horizon Work

Claude Code is the first terminal agent built by the same lab that ships the model. It treats your repo as the work surface, hands its tool calls through MCP servers you configure, and is comfortable working unsupervised for tens of minutes at a stretch. The 87.6% SWE-bench Verified score (Opus 4.7, May 2026) is the highest published benchmark of any agent in the league.

What the benchmark misses: Claude Code is the only major agent built around Anthropic's tool-use protocol from day one, so the MCP ecosystem is native, not an add-on. That makes domain-specific harnesses — your design system, your test runner, your deploy script — cheap to wire in. The cost is the cost: $100–200/month for a developer running Opus heavily.

Aider is the open-source elder of terminal-first coding. Bring your own API key for any frontier model, ask for an edit, and Aider produces a real git diff. The Polyglot benchmark — 225 hard Exercism problems across six languages — exists because Paul Gauthier wanted to test whether Aider was actually good at the languages it claimed. Its scaffolding scores roughly on par with much fancier frameworks on the same model.

Aider's autonomy is bounded by design: it edits, it commits, it asks. Long-horizon planning isn't its thing. But the model-agnosticism and the git-diff transparency make it the obvious tool for engineers who want to compare GPT vs. Claude vs. Gemini on the same task, this week, on their own code.

Pick When Choose Claude Code for long-running refactors, MCP-heavy custom workflows, and teams already invested in the Anthropic ecosystem. Choose Aider when you want model neutrality, transparent diffs, the lowest possible spend, and the cleanest answer to "which model is better on my code today."

The Autonomy Battle · Long Leash vs. Tight Leash

Devin is the maximalist. File a ticket in Slack; Devin spins up a virtual workstation with a real browser, shell, and editor, plans the work, executes, runs tests, opens a PR, and pings you for review. Everything happens on Cognition's infrastructure. The pitch is "junior engineer," not "pair programmer" — Devin is supposed to replace the writing, not augment it.

The 60.8% SWE-bench Verified score is below the model leaders but the only score on the page that was earned end-to-end in a fully autonomous loop with no human in the chair. Independent reviews still split sharply on Devin's PR quality — wins on well-scoped tickets, struggles on anything requiring deep tribal-knowledge context. The price ($500+/month and up) means this is not a developer subscription, it's a head-count line item.

Cline takes the opposite bet. The autonomous loop runs inside VS Code with an explicit approval gate at every step. The agent forms a plan, requests permission to read a file or run a shell command, you see the diff, you say yes or no, and the loop continues. BYO API key against any model. MCP-native. Source on GitHub.

The difference matters more than the spec sheet suggests. With Cline you learn what the agent is doing and why — you can see the prompt, watch the chain, intercept early. With Devin you mostly see the output. Cline is "agent as glass box"; Devin is "agent as service contract." Both are autonomy, only one is observable to you.

Pick When Choose Devin for well-scoped tickets you don't want to write — small bug fixes, scaffolding, the kind of work that bills hourly. Choose Cline when you want autonomous behavior but you also want to understand it, audit it, and switch models without changing tools. Choose neither when the task requires cross-team context that lives in someone's head, not your repo.

Two cautions before reading these scores as gospel. First, every benchmark on this page rewards a slightly different skill. SWE-bench Verified rewards model + harness on real Python bug fixes. Aider Polyglot rewards diff-based editing across six languages. MLE-bench rewards multi-day data-science autonomy. A team that ships React UI components every day cares most about something none of these benchmarks directly test.

Second, scaffolding is half the score. In early 2026, three independent frameworks running the same model scored roughly 17 issues apart on a 731-problem SWE-bench split. The "model" line on a leaderboard quietly absorbs all the harness choices — tool definitions, retrieval strategy, prompt format, retry policy — that the framework author made. Pick a model AND an agent. A great model in a weak harness loses to a mid model in a strong one.

Section 08 · The Philosophy

Three Schools, One Disagreement

Once you sort the field by deployment surface, the disagreement isn't about model quality. It's about who is supposed to be in the chair. Every agent on this page is making a different bet about where coding actually happens, and the bet determines almost everything else — pricing, UX, autonomy ceiling, failure mode.

School A · IDE Copilot

The Agent Lives In Your Editor

The developer drives. The agent suggests at the cursor and works in side panels. Composer or Agent Mode adds multi-file edits without leaving the IDE. The mental model is "pair programmer on your shoulder, not in your shell."

  • +Zero context switch — work stays in one window.
  • +Best for inline completion at typing speed.
  • +Familiar to anyone who's ever used VS Code.
  • −Long-horizon work strains the UX.
  • −Terminal-heavy workflows feel like swimming upstream.
Cursor GitHub Copilot Windsurf

School B · Terminal Native

The Agent Lives In Your Shell

The conversation IS the workflow. The repo is the work surface, MCP servers are the tools, edits are applied as git diffs. Long-horizon, multi-step tasks are first-class — the agent expects to work for tens of minutes unsupervised. The mental model is "junior engineer with a terminal."

  • +Long-horizon planning is the design point.
  • +MCP makes domain-specific tools cheap to wire in.
  • +Native to ops, scripting, devops workflows.
  • −Less suited to UI-heavy frontend work.
  • −Steeper UX learning curve for IDE-first developers.
Claude Code Aider OpenAI Codex CLI

School C · Fully Autonomous

The Agent Lives On A Server

You file a ticket; the agent plans, browses, codes, tests, opens a PR. The developer reviews instead of types. Maximum leash length — and maximum failure surface. The mental model is not "pair programmer" at all; it's "asynchronous contractor you direct in English."

  • +Truly parallelizes — agent works while you sleep.
  • +Best fit for well-scoped tickets.
  • +Lowest human-time cost per task that lands.
  • −Highest dollar cost per task.
  • −Quality variance high; review burden real.

Section 09 · What To Watch In 2026

Players And Shifts Worth Tracking

01

OpenAI Codex CLI

OpenAI's answer to Claude Code shipped as a terminal-first agent in 2025 and has been quietly catching up on the SWE-bench leaderboard. Expect aggressive feature parity with MCP-style tool calling and tighter ChatGPT plan integration in late 2026.

02

Gemini Code Assist

Google's coding assistant now ships inside Android Studio, Firebase, and an expanded Gemini CLI. The IDE integration in JetBrains tools is the catch-up shot; the cloud-side BigQuery + Vertex story is the differentiator.

03

Async PR-Bot Agents

A wave of agents that live inside the PR layer — Copilot Workspaces, Cursor BugBot, Cosine, Sweep — are starting to handle small tickets end-to-end without a developer ever opening an editor. The "junior engineer as service" pattern is generalizing.

04

On-Device & Air-Gapped

Qwen 2.5 Coder, DeepSeek R1, and local Llama-Coder variants closed the gap fast enough that on-device and air-gapped deployments are now viable for finance, defense, and healthcare. Expect Windsurf and Cline to lead the on-prem story.

05

Agent Marketplaces

The MCP server ecosystem started as a developer-utility category and is rapidly becoming a marketplace — domain-specific harnesses for SaaS APIs, design systems, and internal tools sold as installable modules. Anthropic's MCP registry is the early winner.

06

Benchmark Fatigue

SWE-bench Verified is approaching ceiling. Expect 2026 to be the year new harnesses — Terminal-Bench 2.0, MLE-bench Pro, SWE-bench Multimodal, multi-week project benchmarks — start being cited more often than the old standby.
