One protocol. Every tool you'll need.

MCP is USB-C for LLMs — Anthropic's open spec that lets any model reach any tool, file system, database, or SaaS through one uniform interface. The N×M integration problem, collapsed to N+M.

The architecture

Claude at the center.
Every service one beam away.

A client (Claude Desktop, Cursor, an agent) speaks MCP to one or more servers. Each server wraps a tool, a data source, or a workflow. Beams travel both directions: model intent out, structured results back.

Client → Server (out). The model issues a tools/call or resources/read. The protocol is plain JSON-RPC 2.0 over either stdio (local spawn) or HTTP+SSE (remote).

Server → Client (back). Each server publishes its capabilities — what tools exist, what resources can be read, what prompts it provides. The model discovers and uses them at runtime, no glue code per integration.

The three primitives

Tools, resources, prompts.
That's the whole spec.

A server exposes any combination of three primitives. Clients discover them, then the model decides which to invoke. The simplicity is the point — and the reason MCP took off where prior tool-calling specs stalled.

01 · Tools

Things the model can do.

Side-effecting functions the LLM can invoke — create_issue, send_email, run_query. Each tool declares a JSON-Schema for its arguments so the model can call it correctly the first time.

02 · Resources

Things the model can read.

Read-only data the server exposes — a file at file://repo/README.md, a row in a database, a fresh API response. Resources give the model context without giving it write access.

03 · Prompts

Workflows the server provides.

Reusable prompt templates the server ships — /git-summarize, /code-review. The user picks one and the server returns a structured starting prompt the model can then act on.

What it looks like

Ten lines of Python.
That's a server.

A minimal MCP server using the Python SDK. One @server.tool() decorator declares a callable. Claude can now use it.

weather_server.py
from mcp.server import Server
from mcp.server.stdio import stdio_server

server = Server("weather")

@server.tool()
async def get_forecast(city: str) -> str:
    """Return today's forecast for a city."""
    return await fetch(f"https://api.weather.gov/{city}")

if __name__ == "__main__":
    stdio_server(server).run()
