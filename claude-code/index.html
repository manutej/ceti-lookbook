SECTION 01 · WATCH IT THINK

You type what you need. Claude figures out the steps.

The black box on the left is a terminal — the same kind of text-in, text-out window your IT folks live in. But what's happening inside it is new. You're not typing commands. You're typing intent.

Claude reads the situation, lays out a plan, and starts executing — scanning, finding, generating. You watch each step land. If a checkmark looks wrong, you intervene. Most of the time you don't have to.

The skill isn't typing. The skill is knowing what to ask for.

SECTION 02 · WHAT IT ACTUALLY DOES

Six jobs that used to need an engineer.

Claude Code isn't a chatbot with extra steps. It has hands. It edits real files on your computer, runs real commands, and reads real codebases. Below: the six things business teams actually use it for.

01 WRITE & EDIT FILES

Build a working app from a paragraph.

Describe a tool you wish existed — a client report generator, an inventory dashboard, a webhook router — and watch Claude write the files for it across folders. It plans, drafts, tests, fixes its own mistakes, and shows you the diff before saving anything.

02 RUN COMMANDS

Press play on the boring stuff.

Installs, deploys, migrations, backups — Claude runs them, reads the output, and tells you what happened in plain English.

03 DEBUG

"It broke. Make it work."

Paste an error message. Claude traces the cause, names the file and line, suggests a fix, and offers to apply it.

04 DEPLOY

From draft to live URL.

"Put this on Vercel." Claude configures the project, pushes it, and reports the production URL back. Same workflow for Netlify, Cloudflare, or your own server.

05 SEARCH ACROSS CODEBASES

Find a needle in 10,000 files.

"Where do we set the trial length to 14 days?" Claude reads the whole project and points to the exact line — no grep skills required.

06 CHAT + DO

A conversation that leaves footprints.

You and Claude talk back and forth like a Slack thread — but every reply can ship as a file, a commit, a deployment, a sent email. Think of regular Claude as a brilliant assistant who can only point. Claude Code can pick things up.

SECTION 03 · WHERE IT LIVES

The terminal is just a place where text goes in and out.

If "the terminal" sounds intimidating, here's the secret: it's just a chat window. You type, something replies, you read the reply, you type back. The format hasn't changed since 1971.

The difference with Claude Code is that the thing on the other end of your typing isn't a script anymore. It's an intelligent partner that can take real actions on your behalf. So the same window that used to be hostile becomes the most flexible workspace you've ever had.

You can also use Claude Code in VS Code, your browser, or a desktop app if the terminal is too spartan. The same engine, just dressed up.

The metaphor: if email is talking, Slack is chatting, and ChatGPT is asking — Claude Code is delegating. It has hands, and it uses them.
EMAIL Send a request, wait, read a reply. Asynchronous.
SLACK Chat back and forth. Synchronous, words only.
CHATGPT / CLAUDE Ask a smart friend. They reply with words and maybe a draft.
CLAUDE CODE Same chat — but the friend now has hands, a filesystem, and a deploy button.

SECTION 04 · REAL WORKFLOWS

Three people, three Tuesday afternoons.

None of these people identify as engineers. They learned the basics in an afternoon. Here's what they built.

THE FOUNDER

Prototype an investor demo overnight.

Maya had a pitch on Monday and no product yet. She described her idea — a referral tracker for fitness studios — to Claude Code, slept while it built a working version, woke up to a live URL she could click. She fixed the colors herself.

claude "build a referral tracker web app where studios can paste a list of clients and get a shareable referral link per client. deploy to vercel."

THE OPS LEAD

Refactor a messy data pipeline.

Devon inherited a Python script that nobody understood. He asked Claude Code to read it, explain it in plain English, then rewrite it cleanly. The new version is half as long and has tests. Devon now actually knows how it works.

claude "read pipeline.py, explain what each function does in plain language, then refactor for clarity. add tests."

THE MARKETER

Auto-generate client proposals.

Priya was hand-customizing the same proposal deck 12 times a week. She gave Claude Code her template and a CSV of clients; now a single command renders all twelve, swaps in logos, and saves them as PDFs. She edits, then sends.

claude "for every row in clients.csv, render proposal-template.html with the right name, logo, and price. export as pdf."
